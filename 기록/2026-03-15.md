## 백엔드 API 구조 정리

### 3개 축
| 영역 | 역할 |
|------|------|
| `/api` (구 `/ai`) | 사용자 대상 핵심 SaaS 기능 |
| `/reviews` | 외부 리뷰 데이터 수집/적재 |
| `/place-insights` | 리뷰 → 마케팅 인사이트 변환 계층 |

### /ai 핵심 엔드포인트 (원본 prefix `/ai`)
- `GET /ai/me` - 인증 확인
- `POST /ai/generate` - AI 카피 생성 (핵심)
- `GET /ai/history` - 히스토리 목록
- `GET /ai/history/{id}` - 히스토리 상세
- `DELETE /ai/history/{id}` - soft delete
- `POST /ai/results` - 선택 결과 저장

### 현재 상태
- 인증, 생성, 저장, 히스토리, soft delete, 리뷰 업로드 모두 있음
- SaaS MVP + 데이터 확장 기반까지 완성
