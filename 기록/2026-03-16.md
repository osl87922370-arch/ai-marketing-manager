## 2단계: 리뷰 파이프라인 개발 (커밋: 직후)

`POST /place-insights/{dataset_id}/run` — Excel 리뷰 업로드 후 GPT-4o 인사이트 추출 → 채널별 카피 생성 전체 파이프라인.

### 파이프라인 흐름
```
POST /place-insights/{dataset_id}/run
  { channel: "sns"|"blog"|"ads", variant_count: 1~10 }

  1. DB에서 ReviewRow 최대 50개 조회
  2. GPT-4o JSON 모드 → PlaceInsightContext 추출
       (keywords / features / benefits / pain_points / proofs / objections)
  3. ReviewDataset.extracted_blocks 저장
  4. 채널별 Agent 선택 (SNSAgent | BlogAgent | AdsAgent)
  5. Agent → GPT-4o → variants[] 생성
  6. WorkflowRun DB 저장 (run_id, latency_ms 포함)
  7. RunResponse 반환
```

### 새로 만든 파일
| 경로 | 역할 |
|------|------|
| `schemas/workflow.py` | RunRequest, RunResponse |
| `models/workflow_run.py` | WorkflowRun ORM (pipeline 실행 기록) |
| `repositories/review_repository.py` | get_reviews, save_insights |
| `services/insight_service.py` | extract_insights() — GPT-4o → PlaceInsightContext |
| `agents/base.py` | BaseAgent (추상) + _format_insights() 헬퍼 |
| `agents/sns_agent.py` | 인스타그램 카피 — 헤드라인 15자, 본문 50-80자, 해시태그 5개 |
| `agents/blog_agent.py` | 블로그 카피 — 제목 30자, 본문 200-300자, SEO |
| `agents/ads_agent.py` | 광고 카피 — 헤드라인 15자, 본문 30-50자, 고CTR |
| `orchestrators/marketing_orchestrator.py` | run_pipeline() 전체 조율 |

### 발견 및 수정한 버그
| 버그 | 수정 |
|------|------|
| `upload` — `file.file` 객체 전달 | `await file.read()` bytes로 수정, `async def`로 변경 |
| `upload` — dict를 리스트처럼 순회 | `result["items"]` 추출, dict 키 접근으로 변경 |
| `ReviewDataset` 존재하지 않는 `total_rows` 필드 사용 | 제거, `default_year` 및 `id` 명시 |

| `models/place_insights.py` 잘못된 경로 import | `model.place_insights` → `models.place_insights` 수정 |
